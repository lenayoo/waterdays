# WaterDays Agent Instructions

You are helping me ship WaterDays, a simple hydration tracking app.

## Product goal
Make the app feel minimal, cute, and useful.
The core value is: users can instantly know today's hydration status from widgets.

## Design direction
- iOS-like
- clean
- soft blue
- not too childish
- not too complex

## Typography
- Use iOS system font (San Francisco)
- Make main numbers large and bold
- Keep text minimal
- Improve readability with spacing
- Use clear hierarchy (big number > small label)

## UI improvements
- Make UI bigger and easier to read
- Focus on user-friendly layout
- Center important information
- Reduce clutter
- Ensure tap targets are large enough

## Widget priority
1. Lock Screen widget
2. Home Screen widget
3. App screen polish

## Copy rules
Use short English copy:
- 0 cups: Drink water
- 1-7 cups: Hydrated
- 8 cups: You did it

## Do not add
- monthly stats
- complex charts
- new features before release

## Current release goal
Finish WaterDays before I go to Korea.
Prioritize shipping over perfection.

## Localization
- Ensure Korean and Japanese translations are natural and consistent
- Fix awkward or unnatural expressions
- Keep tone simple and user-friendly
- Maintain consistency across all screens and widgets

## Code quality
- Remove unused code, variables, and files
- Avoid duplicate logic
- Keep the code simple and readable
- Refactor only when necessary (do not over-engineer)

## Release priority
- Focus on finishing and shipping the app
- Do not add new features
- Prefer simple and working over perfect
